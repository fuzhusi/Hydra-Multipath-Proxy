# Hydra Multipath Proxy RFC v1.0

> 基于 Rust 的多链路聚合代理协议设计方案

项目代号：Hydra\
版本：v1.0\
目标：实现类似 ADM 多线程下载 + 多服务器带宽聚合能力的通用网络代理系统

------------------------------------------------------------------------

# 1. 项目背景

传统代理结构：

    Client
     |
    Proxy Server
     |
    Internet

问题：

-   单服务器带宽有限
-   单连接只能利用单线路
-   多台服务器无法叠加带宽

Hydra目标：

              Node A
                 \
    Client ---- Hydra ---- Internet
                 /
              Node B

多个节点共同承载一个逻辑连接。

# 6. 多路径调度算法

每个节点维护：

    NodeInfo {

    bandwidth

    latency

    loss_rate

    load

    status

    }

评分：

    score =
    bandwidth * 0.5
    -
    latency * 0.3
    -
    loss * 0.2

选择最高节点传输。

------------------------------------------------------------------------

# 7. 数据分片

数据：

    AAAA BBBB CCCC DDDD

切割：

    Chunk1 AAAA
    Chunk2 BBBB
    Chunk3 CCCC
    Chunk4 DDDD

发送：

    Node A -> Chunk1

    Node B -> Chunk2

    Node C -> Chunk3

    Node A -> Chunk4

收到：

    Chunk3
    Chunk1
    Chunk4
    Chunk2

重新排序输出。
