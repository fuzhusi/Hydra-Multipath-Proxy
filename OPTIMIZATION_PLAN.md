# Hydra Multipath Proxy 优化方案

## 📊 总体评估

| 维度 | 当前状态 | 主要问题 |
|------|---------|---------|
| 性能 | ⚠️ 需改进 | 无连接池，每请求新建 QUIC 连接 |
| 架构 | ⚠️ 需改进 | 多路径聚合未实现，无故障恢复 |
| 安全 | 🔴 严重 | 无认证，无证书验证，无速率限制 |
| 用户体验 | ⚠️ 需改进 | GUI bug，配置不持久化 |

---

## 🔴 P0 - 立即修复（影响核心功能）

### 1. GUI 添加节点输入框不工作
**位置**: `hydra-client-gui/src/main.rs:582`
```rust
// 当前：每帧重建，用户输入丢失
let mut new_node = String::new();
```
**修复**: 将 `new_node` 移到 `HydraApp` 结构体中

### 2. QUIC 连接复用
**位置**: `hydra-client/src/proxy.rs`
**问题**: 每个请求创建新 QUIC Endpoint 和连接
**修复**: 实现连接池，共享单个 Endpoint

### 3. TLS 证书验证跳过
**位置**: `hydra-client/src/transport.rs`
**问题**: `SkipVerification` 完全禁用证书验证
**修复**: 实现证书固定或 TOFU 模型

### 4. 无认证机制
**位置**: `hydra-node/src/server.rs`
**问题**: 任何人知道地址就能使用
**修复**: 实现 PSK 或 HMAC token 认证

---

## 🟡 P1 - 近期修复（影响性能和稳定性）

### 5. 健康检查阻塞 UI
**位置**: `hydra-client-gui/src/main.rs:134-178`
**修复**: 使用后台 tokio Runtime，非阻塞结果轮询

### 6. 配置不持久化
**问题**: 节点和设置关闭后丢失
**修复**: 加载 `config/default.toml`，支持保存到 `~/.config/hydra/`

### 7. 连接数限制未实现
**位置**: `hydra-node/src/server.rs`
**修复**: 使用 `tokio::sync::Semaphore` 限制并发

### 8. 速率限制缺失
**修复**: 实现基于 IP 的令牌桶速率限制

### 9. 缓冲区优化
**问题**: 每连接 256KB 缓冲区
**修复**: 缩小到 16KB，使用缓冲池

### 10. Packet 零拷贝
**位置**: `hydra-protocol/src/packet.rs`
**修复**: `payload` 改为 `Bytes` 类型

---

## 🟢 P2 - 长期改进（提升质量）

### 11. 多路径聚合实现
**当前**: `Splitter`/`Assembler` 是死代码
**目标**: 实现真正的多节点并行传输

### 12. 故障检测与自动切换
**当前**: 节点 Offline 后无法恢复
**修复**: 实现指数退避恢复检测

### 13. 负载均衡多策略
**当前**: 只有单一评分公式
**修复**: 支持加权轮询、最低延迟、最小连接数等策略

### 14. 系统托盘支持
**修复**: 最小化到托盘，保持代理运行

### 15. 日志优化
**问题**: 逐包日志影响性能
**修复**: 握手阶段用 debug，删除数据转发日志
